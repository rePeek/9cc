# C 类型的语法

C 的类型语法以不必要地复杂而闻名。C 的开发者 Dennis Ritchie 作为合著者的《C 程序设计语言》（通称 K&R）中也写到：“C 语言声明的语法，特别是涉及函数指针的声明语法，有时会受到严厉批评。”

像这样，C 的类型语法是一种连作者也暗中承认不太好的设计。不过话虽如此，只要理解规则，这个语法也并没有那么难。

本章会说明 C 类型语法的读法。通过一步步加深理解，到本章结束时，读者应该能够解读 `void (*x)(int)` 或 `void (*signal(int, void (*)(int)))(int)` 这样的复杂类型。

## 表示类型的图

C 中可表达的类型本身相对简单。为了把类型语法的复杂度和类型本身的复杂度分开考虑，先把语法放在一边，只考虑类型本身。

指针、数组等复杂类型，可以用把简单类型通过箭头连接起来的图表示。例如下面这张图表示“指向 `int` 的指针的指针”。

```text
Pointer --ptr_to--> Pointer --ptr_to--> int
```

日语中会从箭头终点向起点读作“`int` 的指针的指针”。英语中则相反，会沿箭头方向读作 “a pointer to a pointer to an int”。

假设变量 `x` 拥有上图中的类型。对“`x` 的类型是什么？”这个问题，最简洁的回答是“指针”。因为第一个箭头指向的是指针类型。注意，`x` 首先是指针，而不是 `int` 等类型。对“这个指针指向的类型是什么？”这个问题，回答仍然是“指针”，因为沿着一个箭头到达的也是指针类型。最后，对“这个指针指向的类型是什么？”这个问题，回答才是“`int`”。

下面的图表示“指向 `int` 的指针的数组”。数组长度是 20。在实际编译器中，数组长度也会像下图这样，作为表示数组的类型成员来表示。

```text
Array(length: 20) --array_of--> Pointer --ptr_to--> int
```

如果变量 `x` 拥有上图的类型，那么 `x` 是长度为 20 的数组类型，该数组的元素是指针，该指针指向的是 `int`。

函数类型也可以用图表示。下面这张图表示一个函数类型：接收 `int` 和指向 `int` 的指针两个参数，并返回指向 `void` 的指针。

```text
Func
  return_type -> Pointer --ptr_to--> void
  param1      -> int
  param2      -> Pointer --ptr_to--> int
```

最后再看一个更复杂的例子。下面的图表示一种类型：指向一个函数的指针，这个函数接收 `int` 作为参数，并返回一个指针；这个返回的指针指向另一个返回 `int` 的函数。用语言描述很复杂，但画成图后只是比较长，结构本身是简单的。

```text
Pointer --ptr_to--> Func
                      param1      -> int
                      return_type -> Pointer --ptr_to--> Func
                                                            return_type -> int
```

如果变量 `x` 拥有上图的类型，那么 `x` 是指针类型，该指针指向的是函数，该函数的参数类型是 `int`，返回值类型是指针类型，而这个指针指向的是函数，该函数的返回值类型是 `int`。

编译器内部也是用和上图相同的方法表示类型。也就是说，涉及指针、数组、函数的复杂类型，在编译器内部会按照上图相同的顺序，表示为用指针连接简单类型结构体的数据结构。因此，可以说这些图正是类型的真实形态。

## 表示类型的记法

像上面这样用图表示时，类型含义会更容易理解，但每次为了理解类型都画图很麻烦。本节考虑一种更紧凑、但不损害图的易懂性的记法。

只要类型中不包含函数类型，图中的所有方框都会没有分支地串成一串。因此，如果类型中只有指针或数组，只要把图中的类型名原样从左到右写出来，就应该能用文字表示这张图。

考虑具体记法。表示指针的方框用 `*` 表示。表示长度为 n 的数组的方框用 `[n]` 表示，表示 `int` 等内建类型的方框则写类型名。这样，下面的图就可以用字符串 `* * int` 表示。

```text
Pointer --ptr_to--> Pointer --ptr_to--> int

=> * * int
```

因为从箭头起点开始依次出现的是指针、指针、`int`，所以记作 `* * int`。反过来，如果给出 `* * int` 这种记法，也可以画出上面的图。也就是说，这种文本表达是一种能把与图相同的信息紧凑写成文本的记法。

下面的图可以用 `[20] * int` 表示。

```text
Array(length: 20) --array_of--> Pointer --ptr_to--> int

=> [20] * int
```

对于函数，采用 `func(参数类型, ...) 返回值类型` 这种写法。例如下图表示的类型记作 `func(int, * int) * void`。请确认这个记法和图是一致的。

```text
Func
  return_type -> Pointer --ptr_to--> void
  param1      -> int
  param2      -> Pointer --ptr_to--> int

=> func(int, * int) * void
```

最后，下面的图表示的类型是 `* func(int) * func() int`。

```text
Pointer --ptr_to--> Func
                      param1      -> int
                      return_type -> Pointer --ptr_to--> Func
                                                            return_type -> int

=> * func(int) * func() int
```

到目前为止说明的记法，大概是类型文本表示中最直接、最简单的。实际上，编程语言 Go 的类型语法与这里说明的记法完全相同。Go 是有 C 的创造者们参与开发的语言，在 Go 中，他们吸取了 C 的反省点，悄悄改良了类型语法。

## C 类型的读法

本节把上面学到的类型记法和 C 的类型记法对应起来，学习如何解读 C 类型。

C 的类型可以从开头分解为下面 4 个部分。

1. base type

2. 表示指针的星号

3. 标识符，或被括号包围的嵌套类型

4. 表示函数或数组的括号

例如 `int x` 的构成是：base type 为 `int`，没有表示指针的星号，标识符为 `x`，没有函数或数组的括号。`unsigned int *x()` 的构成是：base type 为 `unsigned int`，表示指针的星号为 `*`，标识符为 `x`，表示函数的括号为 `()`。`void **(*x)()` 的构成是：base type 为 `void`，表示指针的星号为 `**`，嵌套类型为 `*x`，表示函数的括号为 `()`。

### 非嵌套类型的读法

如果指针后面跟着的只是普通标识符，那么类型解读相对简单。

如果没有函数或数组的括号，类型表示及其含义如下。含义使用上面说明的、与 Go 相同的记法来写。

| --- | --- |
| `int x` | `int` |
| `int *x` | `* int` |
| `int **x` | `* * int` |

如果有函数括号，base type 和表示指针的星号会表示返回值类型。下面给出例子。

| --- | --- |
| `int x()` | `func() int` |
| `int *x()` | `func() * int` |
| `int **x(int)` | `func(int) * * int` |

可以看到，如果先假设没有函数括号来 parse，得到的类型，也就是 `int x`、`int *x`、`int **x` 的类型，只是接在 `func(...)` 后面。

同样，如果有数组括号，就表示“某类型的数组”。下面给出例子。

| --- | --- |
| `int x[5]` | `[5] int` |
| `int *x[5]` | `[5] * int` |
| `int **x[4][5]` | `[4] [5] * * int` |

例如对于 `int *x[5]` 这个类型，`x` 的类型是长度为 5 的数组，该数组元素是指针，该指针指向 `int`。和函数的情况一样，可以看到“没有数组括号时的类型”接在 `[...]` 后面。

### 嵌套类型的读法

如果指针后面不是标识符，而是括号，那么这个括号表示嵌套类型。存在嵌套类型时，可以分别解析括号内的类型和括号外的类型，然后再组合起来，得到整体类型。

以 `int (*x)()` 这个声明为例来 parse。

对于 `int (*x)()`，如果把最初的整个括号看成一个标识符（这里随便称作 `y`）来解释类型，它看起来就是 `int y()`，所以括号外的类型是 `func() int`。另一方面，考虑括号中的 `*x` 这个类型，它表示 `* ___`。括号中的类型没有写 `int` 等 base type，所以作为类型并不完整；这里用 `___` 表示缺失的 base 部分。

通过分别 parse `int (*x)()` 的括号内外，可以得到两个类型：`func() int` 和 `* ___`。整体类型可以通过把外侧类型嵌入到内侧类型缺失的 `___` 部分来得到。这个例子中，整体类型是 `* func() int`。也就是说，在 `int (*x)()` 这个声明中，`x` 是指针，该指针指向函数，该函数返回 `int`。

再举一个例子。`void (*x[20])(int)` 中，如果把最初的括号视为标识符来看，就和 `void y(int)` 是相同模式，所以括号外表示 `func(int) void` 类型。括号中的 `*x[20]` 表示 `[20] * ___`。组合这两个类型后得到 `[20] * func(int) void`。也就是说，`x` 是长度为 20 的数组类型，其元素类型是指针，该指针指向接收 `int` 的函数，该函数返回 `void`。

使用这里说明的方法，可以读取任何复杂类型。作为一个极端例子，读一下 Unix 的 `signal` 函数类型。`signal` 函数以乍看完全不知道是什么的类型而闻名。下面是 `signal` 函数的声明。

```c
void (*signal(int, void (*)(int)))(int);
```

这种复杂类型也可以通过分解读取来解读。首先把最初括号的内部和外部分开考虑。如果把最初的括号视为标识符来考虑类型，它和 `void y(int)` 是相同模式，所以括号外侧表示 `func(int) void` 类型。

括号中是 `*signal(int, void (*)(int))` 这个类型。它的构成是：没有 base type，表示指针的星号是 `*`，标识符是 `signal`，有一组表示函数的括号，并写有两个参数。因此，大致类型是 `func(参数1, 参数2) * ___`。考虑参数类型如下。

- 第一个参数是 `int`。其类型就是 `int`。

- 第二个参数是 `void (*)(int)`。它的构成是：base type 为 `void`，括号包围的嵌套类型是 `*`，有一组表示函数的括号。因此，把最初的括号视为标识符时，类型是 `func(int) void`。再考虑括号中的 `*`，它有一个表示指针的星号，所以类型是 `* ___`。最终，第二个参数的类型就是二者组合后的 `* func(int) void`。

把上述参数类型放入 `func` 的参数部分，就得到 `func(int, * func(int) void) * ___` 这个类型。最后把括号内部的类型与括号外部的类型组合，就得到最终结果：`func(int, * func(int) void) * func(int) void`。

也就是说，`signal` 是一个函数，这个函数接收下面两个参数。

1. int

1. `int`

2. 指向“接收 `int` 并返回 `void` 的函数”的指针

`signal` 的返回值类型是指针，该指针指向接收 `int` 并返回 `void` 的函数。

### 专栏：C 类型语法的意图

C 的类型语法是基于这样的想法设计的：“如果让类型写起来和使用时一样，应该会更容易理解。”在这个设计方针下，例如 `int *x[20]` 这个声明，意味着决定 `x` 的类型，使得写出 `*x[20]` 这个表达式时，其类型为 `int`。也可以说，这等同于解答“在 `int foo = *x[20]` 中，什么样的 `x` 类型能让类型匹配？”这个问题。

如果把 `int *(*x[20])()` 这个声明看成“决定 `x` 的类型，使得 `int foo = *(*x[20])()` 不产生类型错误”的问题，那么可以这样思考：`x` 首先必须是数组；该数组的元素应该可以被 `*` dereference；又因为对这个指针进行了函数调用，所以指针指向的对象是函数；又因为对返回值执行了 `*` dereference，所以它是返回指针的函数。

C 的类型语法看起来有些荒谬，但作为设计方针本身是说得通的。不过，最终还是不得不说它变成了糟糕设计。
